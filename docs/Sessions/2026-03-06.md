## Session 2 -- Play Mode Revert + HUD Minimap

### Work Completed

**Play mode revert/keep choice (bug fix + feature)**
- **Root cause:** `PlayModeController.Enter()` used shallow list copies for the editor state snapshot. `SyncEntityRenderState()` mutated the same objects during play, so killed enemies and picked-up items persisted after exiting play mode.
- **Fix:** Added `DeepCopy()` methods to Entity, MapLayer, MapData, TileGroup, MapDocumentState. Snapshot now uses deep copies.
- **Feature:** `Exit(bool revert)` parameter + ConfirmDialog on F5 exit: "Revert map to pre-play state?" Y = restore snapshot, N = keep play-mode changes.
- **Tests:** 5 new deep copy tests (1528 total, 0 failures).

**HUD minimap**
- Added `Minimap.DrawInRect()` for rendering into a given rectangle (aspect-ratio-correct, centered).
- SidebarHUD reserves adaptive space at the bottom for the minimap. Height computed from map aspect ratio, clamped between 60px and 40% of remaining space. Log shrinks to accommodate.
- Canvas minimap hidden during play mode (redundant with sidebar version).
- Wired through PlayModeController with Minimap reference from MapCanvas.

**Ctrl+S investigation**
- Traced both Ctrl+S (InputRouter:54) and File->Save (MenuActionDispatcher) -- both call identical `_projectManager.Save()`. No code-level bug found. User will monitor for reproduction.
